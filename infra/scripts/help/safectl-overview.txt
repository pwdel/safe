Usage:
  ./safe [options] <group> <command> [args...]
  ./safe help [topic]

Topics:
  overview   General usage and examples
  local      Local Multipass/VM commands
  remote     Remote VPS commands
  terraform  Terraform provisioning commands
  doctl      DigitalOcean CLI passthrough with shared auth env loading
  access     Layered shell access model
  check      Host preflight checks

Global options:
  --host <ip-or-hostname>       Remote host for remote commands
  --ssh-user <user>             SSH user for remote commands (default: operator)
  --bootstrap-user <user>       SSH user for remote bootstrap (default: root)
  --port <port>                 SSH port (default: 22)
  --key <path>                  SSH private key path (default: ~/.ssh/id_ed25519)
  --auto-approve                Use terraform apply -auto-approve
  -h, --help                    Show this overview

Examples:
  ./safe local bootstrap
  ./safe local codex-login
  ./safe check host
  ./safe local test
  ./safe help local
  ./safe --host 203.0.113.10 remote bootstrap
  ./safe --host 203.0.113.10 remote runner-shell
  ./safe terraform deploy --var-file=terraform.tfvars
  ./safe doctl compute droplet list

Notes:
  - terraform and doctl commands auto-load ~/.keys/safe/terraform.env when present.
  - If TF_VAR_do_token is set, safectl also exports DIGITALOCEAN_ACCESS_TOKEN for doctl.
  - remote commands can reuse SAFE_TARGET_HOST from ~/.keys/safe/terraform.env when --host is omitted.
  - Optional task-run settings can be stored in ~/.keys/safe/task-spec.env.
